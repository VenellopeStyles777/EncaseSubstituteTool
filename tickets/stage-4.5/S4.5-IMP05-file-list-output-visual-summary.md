## File-List JSON Contract

Add a stable schema, for example `stage4_5.file_list.v1`.

`file-list.json` must be honest for both successful and unavailable parser states:

- if root listing is real-parser-backed or otherwise available, copy/normalize those entries into a file-list artifact;
- if root listing is unavailable, dependency-unavailable, not-run, or unsupported, write a structured file-list artifact with `entry_count: 0`, status explaining why, warnings, and provenance context;
- never invent file entries or counts.

Required top-level fields:

- `schema_version`
- `status` with `code`, `ok`, and `message`
- `case_id`
- `evidence_id`
- `source_path`
- `redacted_source_path` or redaction metadata when `--redact-paths` is used
- `volume_id`
- `volume_offset`
- `volume_length`
- `filesystem_type`
- `adapter` with name, availability, dependency where available
- `directory_path`
- `parser_backing`
- `entry_count`
- `entries`
- `warnings`
- `read_only_asserted`
- `source_modified`

Each entry must preserve:

- `case_id`
- `evidence_id`
- `source_path`
- `volume_id`
- `volume_offset`
- `volume_length`
- `filesystem_type`
- `adapter_name`
- `file_id`
- `path`
- `name`
- `entry_type`
- `size`
- `allocated`
- `deleted`
- `timestamps`
- `read_only`
- `status`
- `warnings`

Path redaction policy:

- Local JSON artifacts may retain full local paths because `.test-artifacts/` is ignored.
- `command-summary.txt` and `summary.html` should honor `--redact-paths`.
- If `file-list.json` includes local paths while redaction is enabled, add a clear field such as `redaction_applied_to_shared_outputs: true` rather than pretending JSON is redacted.

## CSV Contract

`file-list.csv` must be UTF-8 and use the standard CSV writer, not hand-built string joining.

Required header order:

```text
case_id
evidence_id
source_path
volume_id
volume_offset
volume_length
filesystem_type
adapter_name
file_id
path
name
entry_type
size
allocated
deleted
created
modified
accessed
metadata_changed
status_code
status_ok
status_message
warning_codes
```

Rules:

- Escape commas, quotes, newlines, and formula-looking values safely.
- Join warning codes with `;`.
- Preserve empty values as empty CSV cells.
- Keep JSON as the authoritative artifact; CSV is for quick spreadsheet review.
- For unavailable parser states, write the header row and no data rows unless there are real entries.
