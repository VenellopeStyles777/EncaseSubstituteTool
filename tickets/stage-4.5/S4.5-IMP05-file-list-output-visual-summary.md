# S4.5-IMP05 - File List, Output Bundle, And Visual Summary

Status: Draft

Stage: Stage 4.5 - first testing implementation runway

Owner: VS Code coding agent

## Objective

Implement the inspectable first-testing output bundle that Stage 4.5 was created to reach: file-list JSON/CSV, stronger command summary/artifact inventory, and optional static HTML summary for visual review.

This is still not Stage 5 search/timeline and must not become indexing, saved search, timeline, UI, or reporting infrastructure.

- `tickets/stage-4.5/S4.5-T06-file-list-and-output-plan.md`

- Existing filesystem entry/listing result shapes

## Required Behavior

Implement first-testing artifacts when upstream parser/listing support exists:

- `<output>/segment-discovery.json`
- `<output>/metadata.json`
- `<output>/verification.json`
- `<output>/ewf-stream.json`
- `<output>/volumes.json`
- `<output>/filesystems.json`
- `<output>/file-list.json`
- `<output>/file-list.csv`
- optional `<output>/reports/summary.html`
- updated `run-manifest.json`
- updated `command-summary.txt`

If upstream behavior is unavailable, represent it in manifest/unsupported sections rather than fake files or fake counts.

File-list JSON must preserve:

- case id;
- evidence id;
- source path;
- volume id;
- volume offset and length;
- filesystem type;
- adapter name;
- file id/path/name;
- entry type;
- size;
- allocation/deleted state;
- timestamps;
- status;
- warnings;
- read-only assertion.

CSV must have stable headers and escaping.

HTML, if implemented, must:

- be static local output;
- avoid network assets;
- escape user-controlled text;
- clearly label ok/partial/unavailable/unsupported/failed/skipped states;
- avoid becoming a full UI/search/timeline.

## Tests

Future implementation should cover:

- file-list JSON provenance fields;
- CSV header order and escaping;
- summary includes dependency/unsupported states;
- redaction mode affects summary/HTML shared text;
- HTML escapes metadata and file names;
- no output writer writes beside evidence;
- unavailable parser states still produce honest manifest/summary;
- no search/timeline/index artifacts are created.

## Acceptance Criteria

- A reviewer can inspect a first-testing run visually and via JSON/CSV without opening source code.
- File-list export remains a first-testing artifact, not a search index.
- Unsupported and dependency-unavailable states stay visible.
- Private paths/metadata are redacted in shared summaries when requested.
- No Stage 5 search/timeline, UI, or reporting system is added.
