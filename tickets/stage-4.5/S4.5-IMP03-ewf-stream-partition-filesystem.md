## Required Implementation Path

Implement the smallest real-E01 path that can support a root listing:

1. Segment discovery chooses the first `.E01` and sibling segments.
2. An EWF-backed read-only image stream opens the logical image through `pyewf` or an explicitly verified equivalent.
3. Stream metadata reports logical media size and read-only status.
4. Partition/volume discovery attempts a real parser path or an explicitly documented whole-image fallback when no partition table is present.
5. Filesystem parsing uses a real parser-backed adapter, expected to be `pytsk3` if available.
6. Root listing maps parser entries into the existing `FilesystemEntry` and `list_directory()` result shapes.
7. First-testing command writes status/artifact JSON and summarizes the result.

Keep these boundaries separate:

- EWF metadata/verification remains `PyewfEwfReaderAdapter`.
- EWF byte access belongs to an `EwfImageByteStream` or equivalent `ImageByteStream`.
- Partition/volume parsing produces `VolumeInfo`/`VolumeDiscoveryResult` only.
- Filesystem parsing produces `FilesystemResult`/`FilesystemEntry` only.
- Selected-file content providers for preview/export/hash/signature remain S4.5-IMP04.

## Required Behavior

### EWF Stream

- Add an `EwfImageByteStream` or equivalent read-only stream boundary.
- Do not treat `.E01` as a raw local file.
- Use the discovered segment set; preserve the selected `.E01` as the visible source path.
- Use `stream_type="ewf"` or another explicit EWF label.
- Implement `describe()` with `ImageStreamInfo`.
- Implement bounded `read_at(offset, length)` with `ImageReadResult`.
- Reject negative offsets/lengths before reader calls.
- Handle zero-length reads, read beyond EOF, and truncation at EOF consistently with `LocalFileImageStream`.
- Close native handles deterministically.
- Return structured dependency/open/read/size errors without raw tracebacks.

### Partition/Volume Boundary

- Extend volume discovery only as far as needed for the demo.
- Preserve current `whole_image` behavior for existing tests.
- Add an `auto` or `partition_table` path if practical.
- Preserve byte offsets and lengths in project result shapes.
- If parser units are sectors, convert to bytes only when sector size is known.
- Return separate statuses for stream unavailable, partition dependency unavailable, no partition table, parse error, partial parse, and success.

### Filesystem Root Listing

- Upgrade or wrap `Pytsk3FilesystemAdapter` so it can parse a real root directory when `pytsk3` and an image stream are available.
- Preserve `FilesystemResult`, `FilesystemEntry`, and `list_directory()` shapes.
- Entries must include provenance: source path, volume id, volume offset, volume length, filesystem type if known, adapter name, read-only assertion, and status/warnings.
- Root entries must be parser-backed. Do not use `StubFilesystemAdapter` for a claimed real E01 listing.
- Nested traversal and file bytes are out of scope unless needed minimally by the parser API to enumerate root.
