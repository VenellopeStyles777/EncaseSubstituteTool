## Current Provenance Contract

The selected-file content path may claim `source_kind="real_parser"` only when all of these are true:

- the source entry came from `Pytsk3FilesystemAdapter`, not `StubFilesystemAdapter`;
- the entry has parser-backed status and read-only provenance;
- the entry is a regular file, not a directory or special entry;
- the entry is allocated and not deleted, unless a future reviewed adapter proves recoverable bytes;
- bytes are read through the EWF-backed image stream plus `pytsk3` filesystem/file APIs;
- no stub provider bytes, generated fixture bytes, preview-rendered text, exported file bytes, or local raw `.E01` segment bytes are used as the source.

If any of these cannot be proven, return an explicit non-ok status such as `content_source_unavailable`, `metadata_only_source`, `path_not_file`, `deleted_recovery_unsupported`, `file_too_large_for_in_memory_provider`, or `file_content_unreadable`.

## Required Implementation Path

Add the smallest shared content reader and wrappers needed for this ticket. Suggested names are flexible, but keep the boundaries clear:

1. Shared reader, for example `E01SelectedFileContentReader`:
   - accepts an EWF image stream, a `VolumeInfo`, and a parser-backed `FilesystemEntry`;
   - opens the filesystem at the selected volume offset through `pytsk3`;
   - opens the selected regular file by stable parser identity when possible, such as the inode/address embedded in the S4.5-IMP03 `file_id`;
   - reads bounded ranges and, for explicitly allowed small files only, full content;
   - closes native/parser handles deterministically;
   - returns structured content-read results with status, bytes, source size, bytes read, parser/source identity, provenance, warnings, and read-only assertion.
2. Preview wrapper:
   - implements the existing `PreviewContentProvider` protocol;
   - returns bounded parser-backed bytes to `preview_file()`;
   - never returns stub bytes while claiming real E01 content.
3. Export wrapper:
   - implements the existing `ExportContentProvider` protocol;
   - returns full parser-backed bytes only when the file size is within the explicit in-memory export limit;
   - reuses `export_file()` so destination safety, manifest writing, SHA-256, and byte-count verification stay centralized.
4. Analysis wrapper:
   - implements the existing `AnalysisContentProvider` protocol;
   - reuses `hash_file_content()` for allowed small-file hashes;
   - reuses `detect_file_signature()` for bounded-prefix signature detection.
5. First-testing integration:
   - add explicit selected-file command options, such as `--selected-file-id`, `--selected-file-path`, or a similarly clear form;
   - if no selected file is requested, write honest not-run selected-file artifacts and do not auto-select/export/hash arbitrary evidence files;
   - if a selected file is requested, run the selected-file preview/export/hash/signature path and record separate statuses.

The implementation may support only root entries in this ticket if nested traversal would expand scope. If nested paths are not supported yet, return `path_unsupported` or `content_source_unavailable` rather than faking traversal.

## Size And Safety Policy

Use a bounded first-testing policy unless you implement reviewed streaming support in the same ticket.

Required defaults:

- preview: bounded only, using the existing preview limit unless the command exposes an explicit smaller safe override;
- signature: bounded-prefix only, using existing signature max-byte behavior;
- hash: full-file only when the selected file is at or below a documented in-memory limit;
- export: full-file only when the selected file is at or below the same documented in-memory limit and an explicit output directory is supplied;
- files above the limit return `file_too_large_for_in_memory_provider` for hash/export instead of loading into memory;
- command summaries and JSON artifacts must show whether each operation was `ok`, `not_run`, `refused`, `unavailable`, `unsupported`, or `partial`.

Pick a conservative default limit, document it, and make it visible in artifacts. Do not silently load large real E01-extracted files into memory.
