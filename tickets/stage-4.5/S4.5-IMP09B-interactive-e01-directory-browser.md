# S4.5-IMP09B - Interactive E01 Directory Browser

Status: Done

Stage: Stage 4.5 - first testing implementation runway extension

Owner: VS Code coding agent

Reviewer: Research/review agent

## Objective

Add a live command-line directory browser for the real E01-backed filesystem demo.

S4.5-IMP09 and S4.5-IMP09A prove that the current backend can list one explicit or bounded-demo nested directory through the real parser-backed path. That is enough for artifact proof, but the user also wants a hands-on experience closer to:

```text
dir
cd <folder>
dir
cd ..
exit
```

This ticket should implement that interactive browsing layer while reusing the already reviewed EWF stream, volume discovery, filesystem adapter, and `list_directory()` behavior. It must not become recursive crawling, search/timeline indexing, file-content extraction, or a UI/report feature.

## Architecture Direction

Use the existing directory navigation as the framework.

The existing path is stateless and artifact-focused:

```text
input E01 -> EWF stream -> selected volume/filesystem -> list_directory(volume, path, adapter) -> JSON/CSV artifacts
```

The new browser should be stateful and human-facing:

```text
input E01 -> EWF stream -> selected volume/filesystem -> browser session current_path
browser command -> resolve requested path -> list_directory(volume, requested_path, adapter) -> console rows/status
```

Do not fork a second parser implementation. The browser should call the same parser-backed directory listing boundary that S4.5-IMP09/09A reviewed. If setup code is duplicated, factor a small shared helper instead of creating a new independent EWF/TSK path.

Preferred implementation shape:

- add a small browser/session module such as `app/backend/api/directory_browser.py`;
- expose a CLI, for example:

```powershell
.\.python312-embed\python.exe -m app.backend.api.directory_browser --evidence-dir ".\ Test Image" --first-segment "C16242-1-RL1-E003.E01" --redact-paths
```

If the existing `first_testing` setup helpers make a different entry point cleaner, the coding agent may choose it, but the final command must be documented and copyable.

## Scope Boundary

Allowed:

- create a live command loop that reads commands from stdin and writes human-readable directory output to stdout;
- support the same direct `.E01` and `--evidence-dir` plus `--first-segment` input forms as first testing, or share/validate those input forms through a helper;
- reuse `EwfImageByteStream`, `discover_volumes(..., strategy="partition_table")`, `Pytsk3FilesystemAdapter`, and `list_directory()`;
- keep a current directory path during the session;
- support simple path resolution for absolute paths, relative child names, quoted names with spaces, `.` and `..`;
- display direct child entries for the current directory;
- display status/errors without crashing when a target path is missing, unsupported, unreadable, or a file rather than a directory;
- add automated tests using fake adapters/streams and scripted stdin;
- run a real-image smoke against the ignored local ` Test Image/` set;
- update docs, tickets, prompt indexes, and Stage 5 gate docs so S4.5-IMP09B is required before S4.5-IMP10/S5-T01.

Not allowed:

- no recursive full-volume crawl;
- no broad enumeration or index building;
- no search/timeline work;
- no file-content preview, export, hash, signature, or selected-file auto-selection;
- no deleted recovery/carving;
- no UI/report system/PDF;
- no persistent transcript by default;
- no real evidence commits;
- no internal real filenames or paths quoted in shared handoffs.

## Required Browser Commands

Implement at least:

| Command | Behavior |
| --- | --- |
| `help` or `?` | Print available commands and proof boundaries. |
| `pwd` | Print the current in-image directory path. |
| `ls` and `dir` | List direct child entries of the current directory. |
| `cd <path-or-name>` | Move into a child directory or absolute directory path if parser-backed listing succeeds. |
| `cd ..` | Move to the parent directory without leaving `/`. |
| `cd /` or `root` | Return to filesystem root. |
| `exit` and `quit` | End the browser cleanly with exit code 0. |

Nice-to-have if simple:

- `stat <path-or-name>`: print metadata for a child entry without reading file content.

Do not implement `cat`, `type`, `open`, `preview`, `export`, `hash`, `search`, or recursive `tree` in this ticket.

## Console Output Requirements

The browser should show enough information for a user to see actual files and folders:

- current directory path;
- entry rows with name, type, size when available, allocation/deleted flags when available, and timestamp summary if already available from metadata;
- count summary: total entries, files, directories, other;
- parser/source status: `real_parser_backed`, dependency unavailable, path not found, path not directory, etc.;
- read-only/source-modified assertion in startup or exit summary;
- clear message when a user tries `cd` into a file: it should report `path_not_directory` and keep the current directory unchanged.

Local interactive output may display in-image entry names because that is the purpose of the browser. Shared handoffs and review summaries must report only statuses/counts/type counts and must not quote real filenames, internal paths, metadata values, or file content.

## Testability Requirements

The browser must be testable without manual typing.

Design the command loop so tests can pass an input stream and capture an output stream, or provide an explicit scripted mode. Avoid coupling the whole behavior to raw `input()` calls that cannot be tested cleanly.

Required automated tests:

- command parser accepts `dir`, `ls`, `pwd`, `help`, `cd`, `cd ..`, `cd /`, `root`, `exit`, and `quit`;
- startup lists or can list root through the same `list_directory()` boundary;
- `cd <child-directory>` updates current path only after an `ok` directory listing;
- `cd ..` returns to parent and does not move above `/`;
- `cd <file>` reports `path_not_directory` and does not change current path;
- missing path reports `path_not_found` or equivalent structured status without crashing;
- unavailable parser/dependency states produce a clean message and nonzero or graceful blocked status as appropriate;
- no content extraction, export, hash, search, timeline, recursive crawl, or transcript files are created by default;
- path names with spaces can be handled when quoted.

Run at minimum:

```powershell
.\.python312-embed\python.exe -m pytest app\tests\test_directory_browser.py app\tests\test_directory_listing.py app\tests\test_filesystem_adapter.py
.\.python312-embed\python.exe -m pytest
```

If the coding agent touches `first_testing` helpers or shared parser setup, include the relevant `test_first_testing_command.py` tests in the focused run.

## Real-Image Requirement

Run a real-image browser smoke against the ignored local ` Test Image/` set.

The smoke can be a short scripted stdin session or a carefully described manual session. It must prove:

- command exits cleanly;
- startup/parser setup uses real E01 segment discovery;
- root listing is `real_parser_backed`;
- `dir`/`ls` shows root entries;
- `cd` into at least one real directory succeeds;
- a later `dir`/`ls` shows a nested parser-backed listing with `entry_count > 0`;
- at least one regular file is visible somewhere in the session if available;
- `cd ..` returns to the parent;
- attempting to `cd` into a known file reports `path_not_directory` and does not read content;
- source evidence remains unmodified and read-only is asserted.

The handoff must report only counts/statuses, for example:

```text
Real-image browser smoke exited 0. Root listing real_parser_backed with N entries. Nested navigation ok with N entries, files=N, directories=N, other=N. cd .. returned to parent. File target reported path_not_directory. Source modified false; read-only asserted true.
```

Do not quote the selected real directory name, real child file names, internal paths, metadata values, or file content.

## Implementation Notes

S4.5-IMP09B adds `app.backend.api.directory_browser` and exposes reusable helpers from `app.backend.api`.

The browser:

- accepts the same direct `.E01` and `--evidence-dir` plus `--first-segment` input forms used by first testing;
- builds the existing EWF stream, partition-table volume discovery, `Pytsk3FilesystemAdapter`, and `list_directory()` path;
- keeps current in-image directory state;
- supports `help` / `?`, `pwd`, `ls` / `dir`, `cd <path-or-name>`, `cd ..`, `cd /`, `root`, `exit`, and `quit`;
- handles quoted names with spaces;
- reports `path_not_directory` and keeps the current path unchanged when `cd` targets a file;
- writes no transcript or artifact bundle by default.

Scope remains intentionally narrow: no recursive crawl, broad enumeration, content preview/export/hash/signature, selected-file auto-selection, search/timeline, UI/reporting, deleted recovery, carving, packaging, dependency installation, commit, or push.
