## Required Behavior

### Identity Input

The first-testing CLI must allow:

```powershell
--project-name "Demo Case"
--inspector "Inspector Name"
--custodian "Custodian Name"
```

Required compatibility:

- `--project-name` is the preferred user-facing project/case display name.
- `--case-name` remains supported as an older alias/fallback.
- If both `--project-name` and `--case-name` are supplied, use `--project-name` as the display/project name and keep `--case-name` only if the existing data model still needs it.
- `--inspector` is the preferred examiner/inspector field.
- `--actor` remains supported as the audit actor alias/fallback.
- If both `--inspector` and `--actor` are supplied, use `--inspector` for display/examiner fields and preserve the actual audit actor behavior intentionally.
- `--custodian` is optional and must remain blank/unknown/null when omitted.
- These fields must not be confused with evidence source file paths.

### Artifact Output

The first-testing output must include the supplied identity fields in suitable existing artifacts:

- `outputs/case.json`
- `run-manifest.json`
- `command-summary.txt`
- `outputs/reports/summary.html`

Use a stable JSON shape such as an `identity`, `project`, or `case` section. The exact field placement may follow existing code style, but review must be able to find:

- project name;
- inspector name;
- custodian name;
- compatibility/fallback behavior for omitted fields;
- provenance remains separate from identity.

### Browser And Navigation Labels

The live browser command must stop presenting the root display as if the user is browsing only an `.E01` segment. It may still record the source segment path internally or in provenance/status output, but the interactive display should use a logical image/project label.

Acceptable prompt/header examples:

```text
Demo Case / volume-2 /
Demo Case [volume-2] /
Image: Demo Case /
```

Unacceptable as the primary interactive navigation label when a project name is supplied:

```text
C16242-1-RL1-E003.E01:/
E01:/
```

If no project name is supplied, use a neutral fallback such as `Evidence Image`, `Logical Image`, or an existing safe case label. Do not expose private full local paths when `--redact-paths` is active.

The browser may browse one selected volume/filesystem at a time, but the display must be honest about that. It should not claim whole-image recursive coverage if it is only showing the selected filesystem.
