## Required Guide Sections

The guide must include:

- prerequisites:
  - Python environment for normal tests;
  - project-local portable runtime path `.\.python312-embed\python.exe` for real E01 parser-backed commands;
  - running from repository root;
  - evidence folder requirements;
  - why real evidence must stay ignored/uncommitted;
  - optional dependency notes for `pyewf`/`pytsk3`, including that the portable runtime is the reviewed local path where available;
- safety rules:
  - select `.E01`, not `.E02+`;
  - write outputs only under `.test-artifacts/first-testing/` or another approved non-evidence path;
  - do not write beside evidence segments;
  - use `--redact-paths` for shared console/summary text;
  - do not paste unredacted private paths or metadata without approval;
- commands:
  - basic first-testing command with a direct `.E01` path;
  - `--evidence-dir` plus `--first-segment`;
  - explicit `--case` and optional `--output`;
  - `--json-only`;
  - `--redact-paths`;
  - dependency-free `--adapter stub` smoke check with a clearly safe/non-sensitive test input;
  - local ` Test Image/` no-selection command if that folder exists;
  - selected-file command template with placeholder `--selected-file-id` or `--selected-file-path`, explicitly not using a real file unless approved;
  - focused pytest commands for the first-testing/parser-related tests;
  - full pytest command through the portable runtime;
- expected artifacts:
  - case database;
  - run manifest;
  - command summary;
  - intake/case/audit/unsupported sections;
  - metadata/verification artifacts if S4.5-IMP02 is done;
  - stream/volume/filesystem artifacts if S4.5-IMP03 is done;
  - selected-file artifacts if S4.5-IMP04 is done;
  - file-list/CSV/HTML artifacts if S4.5-IMP05 is done;
- inspection steps:
  - open/read `command-summary.txt`;
  - inspect `run-manifest.json`;
  - inspect `unsupported-sections.json`;
  - inspect file-list/HTML when available;
  - verify `source_modified: false` and `read_only_asserted: true`;
  - confirm outputs are outside evidence paths;
  - confirm selected-file artifacts are `not_run` in the no-selection smoke;
  - confirm shared notes do not expose private metadata, root-entry names, or real file content;
- expected statuses:
  - `ok_with_unsupported_sections`;
  - `metadata_unavailable`;
  - `dependency_unavailable`;
  - `not_run`;
  - `not_supported`;
  - `verification_ok`;
  - `verification_failed`;
  - `unsafe_output_path`;
  - `invalid_input`;
- troubleshooting:
  - missing `pyewf`;
  - missing `pytsk3`;
  - selected `.E02` by mistake;
  - output/case path overlaps evidence;
  - unsupported filesystem/content path;
  - redaction expectations;
- proof boundaries:
  - what each command proves;
  - what each command does not prove;
  - which future ticket owns any missing capability;
  - what the current demo can show at minimum: open a case workspace, ingest an E01 segment set, read real EWF metadata when dependencies support it, open an EWF-backed stream, find volumes, list the root filesystem, export the root file list, and create a static local summary;
  - what remains outside the demo: nested navigation, broad crawl, selected-file real-content run without approval, search/timeline, UI, PDF/report system, deleted recovery, carving, packaging;
- reviewer transcript template:
  - command run;
  - evidence path redacted as;
  - output path;
  - segment count;
  - metadata status;
  - verification status;
  - filesystem/file-list status;
  - source modified;
  - artifacts checked;
  - skipped/unavailable pieces;
  - privacy notes.
