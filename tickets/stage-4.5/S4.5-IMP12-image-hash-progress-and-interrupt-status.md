## Required Direction

The hash progress must be based on actual bytes hashed from the logical EWF image stream and the stream's logical media size. It must not be based on segment container file size, stored EWF hash metadata, estimated file counts, or arbitrary spinner-only output.

## Required Behavior

When `--hash-image` is requested, expose progress that includes:

- bytes hashed so far;
- total logical media size when known;
- percent complete when total is known;
- elapsed seconds;
- current throughput or average throughput;
- estimated remaining time when enough information exists;
- final status.

Progress may be shown as terminal text, a progress bar, and/or an updating `outputs/image-hash-progress.json` artifact. Prefer a terminal progress bar for the user's command-line demo, but keep `--json-only` behavior parseable.

Rules:

- If `--json-only` is active, stdout must remain final parseable JSON. Progress should go to stderr and/or a progress artifact.
- `--redact-paths` must redact evidence paths in progress/status text.
- The completed `outputs/image-hash.json` digest must remain trustworthy and unchanged in meaning.
- A digest must only be present when hashing completes.
- If the user interrupts hashing, write or preserve a non-success status such as `interrupted`, including bytes hashed so far, logical media size, elapsed time, read-only assertion, source-modified assertion, and warnings where available.
- Default runs without `--hash-image` must still write `image-hash.json` as `not_run`.
- Stored EWF hashes and segment-container hashes must not be presented as this completed logical-image hash proof.

## Implementation Notes

- Added logical-image hash progress callbacks based on actual bytes hashed from the EWF logical image stream and the stream's logical media size.
- Added a terminal progress line for `--hash-image` with percent, byte counts, average throughput, ETA, status, and redacted source text when `--redact-paths` is active.
- Kept `--json-only` stdout as the final parseable run manifest; progress is emitted to stderr and written to `outputs/image-hash-progress.json`.
- Added `outputs/image-hash-progress.json` with schema `stage4_5.image_hash_progress.v1`, byte counts, logical media size, percent, elapsed time, throughput, ETA, digest availability, warnings, read-only assertion, and source-modified assertion.
- Preserved completed `outputs/image-hash.json` semantics. A digest is available only when the full logical-image hash completes; interrupted or failed runs omit the digest.
- Added interrupted hash handling so a stopped hash records status `interrupted`, bytes hashed so far, warning `image_hash_interrupted`, and no digest.
- Did not run the local approximately 1 TB full-image hash to completion for this ticket.
