Status: Done

Stage: Stage 4.5 - first testing implementation runway extension

Owner: VS Code coding agent

Reviewer: Research/review agent

## Objective

Correct the S4.5-IMP09 nested-navigation demo so the hands-on path visibly reaches regular files inside the real E01-backed filesystem, not only nested directories.

S4.5-IMP09 added real parser-backed one-directory navigation and produced a valid nested directory listing, but the reviewed real-image demo mode stopped at a nested directory containing only directories: `file_count = 0`, `directory_count = 4`. A reviewer-run explicit deeper path proved the implementation can reach regular files, producing a parser-backed listing with `file_count = 19`, but the default demo command still does not satisfy the user's stated bar: "at the very least see actual files within the image."

This ticket must keep the S4.5-IMP09 parser-backed work and add a narrow correction before S4.5-IMP09 is accepted as done.

## Scope Boundary

Allowed:

- improve `--demo-list-first-directory` so it prefers and returns a bounded parser-backed directory listing that contains regular files when available;
- use a bounded two-level probe from root-directory candidates into their direct child directories;
- record attempt counts, selected depth, and file-visible status in `directory-listing.json`, `navigation-readiness.json`, manifest, command summary, and HTML summary;
- fix explicit `--list-directory-path` behavior so a known file path returns `path_not_directory`, not `path_not_found`;
- add tests and rerun focused/full verification;
- run a real-image smoke that produces a nonzero `file_count` without quoting names or paths in the handoff.

Not allowed:

- no recursive full-volume crawl;
- no broad enumeration or search/timeline index;
- no file content reads, preview, export, hashing, or signature analysis;
- no selected-file auto-selection;
- no deleted recovery/carving;
- no UI/report system/PDF;
- no real evidence commits.

## Required Behavior

Keep these S4.5-IMP09 options:

- `--list-directory-path`;
- `--demo-list-first-directory`.

Enhance `--demo-list-first-directory`:

- first try allocated root-directory candidates as S4.5-IMP09 does now;
- if a candidate listing is nonempty but has `file_count = 0`, inspect that candidate's direct child directories, bounded and non-recursive;
- stop at the first parser-backed nonempty listing with `file_count > 0`;
- if no file-containing directory is found, fall back to the best nonempty parser-backed listing but set readiness/status messaging so it is not claimed as file-visible proof;
- do not scan beyond root candidates plus their direct child directories;
- record `selected_depth`, `file_visible`, `candidate_directory_count`, `attempted_directory_count`, and `attempted_child_directory_count`.

Fix explicit file-path status:

- if `--list-directory-path` points to an existing regular file, including a nested regular file, return structured `path_not_directory`;
- do not report a known existing file path as `path_not_found`;
- do not read file content to make this decision.
